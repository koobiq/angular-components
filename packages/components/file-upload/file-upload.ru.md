Позволяет пользователю загрузить файлы в продукт.

<!-- example(file-upload-single-overview) -->

### Множественная загрузка

В зону загрузки можно перетащить несколько файлов или выбрать их через системный диалог.

<!-- example(file-upload-multiple-default-overview) -->

#### Компактный вариант

Для множественного загрузчика доступен компактный вид в незаполненном состоянии.

<!-- example(file-upload-multiple-compact-overview) -->

### Размер файла

При загрузке нескольких файлов их размер всегда показан в списке, а в одиночном загрузчике показ этого параметра настраивается отдельно.

<!-- example(file-upload-single-with-size) -->

### Иконка для выбранного файла

Выбранным файлам доступна настройка особенной иконки. Например, для изображений.

<!-- example(file-upload-multiple-with-custom-icon) -->

### Выбор папки или файлов

Компонент можно настроить, чтобы была возможность выбирать только файлы, только папки, файлы или папки.

<!-- example(file-upload-allowed) -->

### Неактивное состояние

В неактивном состоянии компонент не пропадает в фокус, не работает механика перетаскивания.

<!-- example(file-upload-multiple-disabled) -->

### Ошибка

<!-- example(file-upload-multiple-error) -->

При выборе невалидного файла состояние ошибки получает только соответствующий элемент в списке, а не весь компонент. Сообщения об ошибке выводятся под загрузчиком

<!-- example(file-upload-multiple-error-filled) -->

### Высота списка выбранных элементов

По умолчанию компонент растет по высоте, если выбрать много файлов. Но можно настроить максимальный размер, после которого появится прокрутка.

<!-- example(file-upload-multiple-with-max-height) -->

Также возможно установить фиксированную высоту загрузчика.

<!-- example(file-upload-multiple-with-fixed-height) -->

### Полноэкранный загрузчик файлов

Пользователь может перетащить файлы на страницу, в этот момент на весь экран будет показана паранджа загрузчика.

<!-- example(file-upload-dropzone) -->

### Область загрузки

Областью для перетаскивания может быть не только весь экран, сам компонент File Upload в форме, но и отдельная часть экрана

<!-- example(file-upload-local-dropzone) -->
