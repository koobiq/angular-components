У каждого языка свои нормы записи чисел: разделитель дробной части, группировка разрядов, обозначение тысяч и миллионов. Форматтеры применяют нормы текущей локали и обновляют запись при переключении языка. Применяйте их везде, где числа видит пользователь.

### Группировка разрядов

В больших числах разряды группируют по три, считая справа. В тексте используйте пайп `kbqNumber`, в таблицах&nbsp;— `kbqTableNumber`. Пайпы отличаются порогом, с которого начинается группировка.

`kbqNumber` группирует разряды в числах от 10&#8239;000, то есть начиная с пяти разрядов: 1234, 56&#8239;789, 987&#8239;654, 1&#8239;000&#8239;000&#8239;000. В тексте короткое число без разделителя читается естественнее.

`kbqTableNumber` отделяет разряды всегда: 1&#8239;234, 56&#8239;789, 987&#8239;654, 1&#8239;000&#8239;000&#8239;000. В таблице ровные столбцы позволяют сравнивать значения по разрядам сверху вниз.

### Форматирование

Оба пайпа принимают необязательную строку `digitsInfo`, которая задает количество целых и дробных разрядов:

```
{minIntegerDigits}.{minFractionDigits}-{maxFractionDigits}-{useGrouping}
```

| Параметр            | По&nbsp;умолчанию | Описание                                    |
| ------------------- | ----------------- | ------------------------------------------- |
| `minIntegerDigits`  | `1`               | Минимальное количество целых разрядов       |
| `minFractionDigits` | `0`               | Минимальное количество дробных разрядов     |
| `maxFractionDigits` | `3`               | Максимальное количество дробных разрядов    |
| `useGrouping`       | `true`            | Включает или выключает группировку разрядов |

Например, `'4.0-2-false'` оставляет 4 целых и до 2 дробных разрядов и отключает группировку.

### Сокращенная запись

Пайп `kbqRoundNumber` отображает большие числа в сокращенном виде: тысячу&nbsp;— как `K`, миллион&nbsp;— как `M`, миллиард&nbsp;— как `B` и так далее.
